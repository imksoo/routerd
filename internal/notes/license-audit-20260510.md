# License audit 2026-05-10

Scope:
- routerd repository source license
- Go module dependencies linked into routerd binaries
- Alpine packages used by the live ISO
- release archive and live ISO notice placement

Conclusion:
- routerd itself remains BSD 3-Clause.
- Go module license scan did not detect GPL, LGPL, or AGPL license text in linked Go module license files.
- `github.com/hashicorp/golang-lru/v2` is correctly detected as MPL-2.0 after checking its LICENSE text.
- Alpine live ISO packages include GPL-family packages such as dnsmasq, nftables, wireguard-tools, ppp, iproute2, iptables, and filesystem utilities.
- These Alpine packages are included as separate components in an aggregate distribution. The live ISO is not treated as one relicensed GPL work.

Artifacts and implementation:
- `LICENSE` contains BSD 3-Clause text for routerd.
- `THIRD_PARTY_LICENSES.md` is generated by `scripts/collect-third-party-licenses.sh`.
- The generated inventory includes Go module source URLs through pkg.go.dev and Alpine package upstream URLs through APKINDEX metadata.
- `make third-party-licenses` regenerates the inventory.
- release archives include `share/doc/LICENSE` and `share/doc/THIRD_PARTY_LICENSES.md`.
- live ISO overlays include `/usr/share/licenses/routerd/LICENSE` and `/usr/share/licenses/routerd/THIRD_PARTY_LICENSES.txt`.
- `docs/legal.md` documents redistribution obligations and release checklist.

Validation performed:
- `./scripts/collect-third-party-licenses.sh THIRD_PARTY_LICENSES.md`
- `shellcheck scripts/collect-third-party-licenses.sh scripts/build-live-iso.sh packaging/install.sh scripts/release.sh`
- `make dist ROUTERD_OS=linux GOARCH=amd64 VERSION=vlicense-test`
- `tar -tzf dist/linux-amd64/routerd-vlicense-test-linux-amd64.tar.gz` confirmed `share/doc/LICENSE` and `share/doc/THIRD_PARTY_LICENSES.md`.
- `make live-iso VERSION=vlicense-test`
- extracted `routerd.apkovl.tar.gz` from the ISO and confirmed `/usr/share/licenses/routerd/LICENSE` and `/usr/share/licenses/routerd/THIRD_PARTY_LICENSES.txt`.

Open risk:
- The license classifier is heuristic. Formal releases still require review when dependency sets change.
- Alpine package metadata is collected for `latest-stable` / `x86_64` by default. Cross-architecture package metadata may differ slightly.
