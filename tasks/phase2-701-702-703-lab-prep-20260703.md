## Azure route-table variant proposal

The proposed variant changes only the Azure SAM leaf capture path:

- add an Azure UDR route table for the SAM subnet;
- associate that route table with the Azure SAM subnet;
- expose the route table ARM ID as `AZURE_ROUTE_TABLE_REF`;
- set each Azure leaf `captureStrategy: route-table`;
- set the route-table next hop to that leaf's NIC primary private IP, not the
  moved `/32`;
- keep secondary-IP strategy unchanged on AWS/OCI and any non-route-table pools.

Before phase 2 execution, the operator must convert the proposal diff into the
active OpenTofu/config generator shape and review it locally. The first
permitted live step is environment certification, not direct product
qualification.
